# W0 范围冻结清单（2026-04-13 ~ 2026-04-15）

## 1. W0 目标
- 完成 StrokeClaw W0 起跑线：`范围冻结 + 交互骨架冻结 + API 契约冻结`。
- 产出可联调骨架，不追求功能完备。

## 2. W0 范围内（In Scope）
- 新增 W0 任务入口页（`/strokeclaw/w0`）：
  - 支持创建 run
  - 展示 `Stroke Orchestration` 计划面板
  - 展示 Runtime 状态占位与事件占位
- 统一 run 增量字段（输出兼容）：
  - `plan_frames[]`
  - `replan_count`
  - `termination_reason`
  - `human_checkpoint`
  - `finalization`
- 统一 events 增量字段（输出兼容）：
  - `event_type`
  - `phase`
  - `node_name`
- 冻结 `POST /api/agent/runs/{run_id}/review` 协议（W0 返回未实现占位）。
- 输出 W0 文档包与 W1 backlog。

## 3. W0 范围外（Out of Scope）
- 算法升级（ICV/EKV/Consensus 逻辑变更）。
- 模型替换或权重策略重构。
- 全站 UI 重构（React 与模板页统一迁移）。
- WebSocket/SSE 实时通道改造（W0 只做轮询）。
- 医生复核业务闭环完整实现（W1 再做动作落库与状态跳转）。

## 4. 成功标准
- 可以从任务入口创建 run，并看到计划面板与状态变化。
- Runtime 页面显示 run 基本状态、当前阶段、当前节点占位信息。
- 事件流按时间顺序展示，状态标签和 token 一致。
- 后端缺失新增字段时，前端有兼容回退，不白屏。

## 5. 依赖与责任
- 前端主责：W0 页面骨架与容错渲染。
- 后端主责：增量字段、事件 token、review 协议占位。
- 产品/架构主责：文档冻结、验收清单与 W1 分解。

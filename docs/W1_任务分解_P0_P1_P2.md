# W1 任务分解（P0 / P1 / P2）

## P0（必须完成）
- 实装 `POST /api/agent/runs/{run_id}/review` 业务动作：
  - `accept/edit/reject/sign/handoff`
  - 动作落状态机与事件流
- 打通 `paused_review_required -> human_review_completed -> resumed` 主链。
- Runtime 页增加待确认项详情卡（来源 `human_checkpoint`）。
- Finalization 对象从占位改为真实状态（签名/回写/版本）。

## P1（强烈建议）
- 统一节点输入/输出摘要字段，避免前端依赖原始 `output_ref` 自解码。
- 加入 `suggestion_generated` 事件类型并在 UI 高亮。
- 增强错误语义：`issue_found` 按风险级别分层。
- 补齐 review 历史列表（操作人、时间、动作、摘要）。

## P2（可选优化）
- 轮询升级 SSE（保持轮询回退）。
- 增加 trace 导出（JSON + 文本双格式）。
- 完善页面动效与演示脚本模式（自动播放流程轨迹）。

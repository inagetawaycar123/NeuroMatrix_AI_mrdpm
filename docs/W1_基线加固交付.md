## 2. 临床工作流真值表（冻结）

### 2.1 仅允许 4 种合法路径（固定）
| ImagingPath | `available_modalities`（原始输入示例） | 归一化后（判定用） |
|---|---|---|
| `NCCT-only` | `['ncct']` | `['ncct']` |
| `NCCT + single-phase-CTA` | `['ncct','mcat']` / `['ncct','vcat']` / `['ncct','dcta']` | `['ncct','mcta']` / `['ncct','vcta']` / `['ncct','dcta']` |
| `NCCT + mCTA` | `['ncct','mcta','vcta','dcta']` | `['ncct','mcta','vcta','dcta']` |
| `NCCT + mCTA + CTP` | `['ncct','mcta','vcta','dcta','cbf','cbv','tmax']` | `['ncct','mcta','vcta','dcta','cbf','cbv','tmax']` |

约束：
1. 路径判定必须落入上述 4 类之一。
2. 不符合上述组合的输入，标记为“非法组合/需人工复核”。

### 2.2 触发规则冻结（固定）
| ImagingPath | `generate_ctp_maps` | `run_stroke_analysis` |
|---|---|---|
| `NCCT-only` | 不触发 | 不触发 |
| `NCCT + single-phase-CTA` | 不触发 | 不触发 |
| `NCCT + mCTA` | 触发（MRDPM 生成 CTP） | 触发 |
| `NCCT + mCTA + CTP` | 不触发（已有真实 CTP） | 触发 |

关键约束：
1. 仅 `NCCT+mCTA`、`NCCT+mCTA+CTP` 触发 `run_stroke_analysis`。
2. 仅 `NCCT+mCTA` 触发 `generate_ctp_maps(MRDPM)`。
3. `NCCT+mCTA+CTP` 直接消费真实 `cbf/cbv/tmax`。

## 3. 字段语义冻结（新增）

### 3.1 `available_modalities`
1. 模态识别主字段。
2. 允许历史原始值进入（如 `mcat/vcat/dcat`）。
3. 判定前必须归一化为 `mcta/vcta/dcta`。

### 3.2 `hemisphere`
1. 偏侧识别主字段。
2. 枚举固定：`right|left|both`。
3. 任何非枚举值都必须在进入判定链路前拒绝或纠正。

### 3.3 原始值/归一化值双轨规则
1. `raw_modalities`：保留原始输入用于追溯。
2. `canonical_modalities`：归一化后用于路径判定与触发逻辑。
3. 路径判定、Tool触发一律以 `canonical_modalities` 为准。

## 4. 状态机（冻结版）

### 4.1 两层状态（禁止混用）
1. 运行状态（CanonicalRunStatus）：`queued/running/succeeded/failed/cancelled`
2. 步骤状态（CanonicalStepStatus）：`pending/running/completed/failed/skipped`

### 4.2 汇总映射（固定）
| 步骤层汇总条件 | 运行层结果 |
|---|---|
| 存在任一步骤 `failed` | `failed` |
| 全部步骤 `completed` 或 `skipped` | `succeeded` |
| 至少一步骤 `running` 且无 `failed` | `running` |
| 任务已创建未启动 | `queued` |
| 被外部中止 | `cancelled` |

## 5. 阶段枚举（冻结版）

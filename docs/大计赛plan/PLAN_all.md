# NeuroMatrix AI 比赛版重排计划（前端惊艳优先，后端渐进补齐）

## Summary
1. 本计划以 `New Agent Plan.md` 为基准，采用“**Cockpit主舞台 + 影像页/报告页双关键落点**”的比赛策略。  
2. 执行方式改为双轨：`展示轨（前端优先）` 与 `能力轨（后端渐进）` 并行，确保“即使后端未全跑通，前端依然可完整演示智能体价值”。  
3. 比赛目标是完成 `P0 全部` + `P1 可演示闭环`，P2仅保留原型与答辩论证材料。  
4. 团队分工保持 6 人结构，但把康/刘/雷的展示与证据叙事任务前置到前 4 周，保证“早出效果、可持续打磨”。

---

## 一、比赛导向执行模型（决策已锁定）
### 1) 评委体验主线
1. 评委先看 `Agent Cockpit`：必须第一眼看到“规划-执行-校验-请求人工确认”的智能体过程。  
2. Agent流程结束后看 `影像页`：必须看到“影像结果与结论的关系”，不是纯图片。  
3. 最后看 `报告页`：必须体现“结构化报告 + 医生复核 + 可追溯证据”。

### 2) 双轨交付
| 轨道 | 目标 | 交付原则 |
|---|---|---|
| 展示轨（A） | 页面交互与叙事惊艳 | 优先实现可视化和高保真交互，允许后端先用mock/混合数据 |
| 能力轨（B） | 后端架构与工具编排 | 逐步把 mock 节点替换为真实工具与模型 |
| 可信轨（C） | 医疗可信与答辩可解释 | 全链路标注证据来源、状态、置信度、是否真实运行 |

### 3) Mock/Real 显式策略（避免“演示造假”风险）
1. 所有节点必须显示 `source_tag`：`real` / `mock` / `hybrid`。  
2. Cockpit页与节点卡片统一展示“数据来源标识”。  
3. 答辩口径明确：比赛版为“演示优先工程化样机”，并说明真实化替换路径。

## 三、公共接口/对象变更（实现时必须统一）
### 1) 对象契约（比赛版最低要求）
| 对象 | 必要字段 |
|---|---|
| AgentRun | run_id, case_id, status, stage, plan_revision, current_step, source_tag |
| AgentStep | step_id, run_id, step_key, status, confidence, latency_ms, source_tag |
| ToolCall | call_id, step_id, tool_name, input_ref, output_ref, retryable, error_code |
| TraceEvent | event_id, run_id, event_type, timestamp, actor, message |
| ClinicalDecisionBundle | triage_result, vessel_result, perfusion_result, stroke_analysis, evidence_items, consistency, report_payload, human_review |

### 2) API最小增量（兼容现有接口）
1. `GET /api/agent/runs/{run_id}/graph`：返回DAG节点与连边、节点状态与来源标记。  
2. `GET /api/agent/runs/{run_id}/steps/{step_id}`：返回节点详情（输入/输出/置信度/耗时/证据）。  
3. `GET /api/agent/runs/{run_id}/decision-bundle`：统一提供报告页与证据页所需数据。  
4. `POST /api/demo/scenarios/{scenario_id}/start`：比赛演示场景启动（真实/混合/mock）。  
5. 现有 `runs/events/result/review/validation` 接口保留，逐步接入新对象字段。

### 3) 前端页面DoD（Definition of Done）
| 页面 | DoD |
|---|---|
| Cockpit | 可实时展示DAG、节点状态、节点详情、证据摘要、来源标记 |
| 影像页 | 至少3类叠加结果可切换，且每个关键结论可定位对应影像证据 |
| 报告页 | 分块编辑、复核操作、版本对比、证据引用可见 |
| 证据页 | 结论→证据映射可展开，显示等级、来源、摘要 |

## 五、测试与验收场景（比赛必测）
1. 路径分流测试：`NCCT-only / NCCT+mCTA / NCCT+mCTA+CTP` 三条路径都能在Cockpit正确展示。  
2. 新模型节点测试：NCCT三分类与血管堵塞三分类能输出置信度并影响路径。  
3. 降级测试：模型失败时自动切换mock，页面不中断，且明确标注 `source_tag=mock`。  
4. 证据追溯测试：每个关键结论可展开证据来源（等级、文档、摘要）。  
5. 复核闭环测试：报告分段确认、修改记录、最终确认可跑通。  
6. 演示稳定性测试：三条Demo连续演示30分钟无阻断故障。

## Assumptions & Defaults
1. 比赛版本采用“**前端惊艳优先**”，后端按渐进策略补齐，不做一次性大重构。  
2. `Cockpit` 是主舞台；`影像页` 与 `报告页` 是关键收尾页面。  
3. P1能力默认做到“可演示、可讲清”，暂不追求完整工程化闭环。  
4. 模型接入默认“真实优先 + mock兜底”，并强制前端显示来源标记。  
5. 每周固定一次里程碑评审与一次端到端彩排，W8前冻结比赛版本。

# 卒中项目 Agent 化每周任务清单（一页版）

用途：给项目经理和开发同学快速执行，不看长文也能推进。  
对应主文档：`docs/Agent_PLAN.md`  
对应执行文档：`docs/卒中先锋_agent落地路线图_执行版.md`

## 一句话总目标
8 周内完成一个“能跑、可解释、可验收”的 Agent 版本，不破坏现有 viewer/report/chat。

## 每周清单（可直接打勾）

### 第1周：统一规则
- [x] 输出术语表（CTA 期相、CTP 参数、AI 报告、临床问答）
- [x] 输出“4路径真值表（冻结）”
- [x] 固化“仅2路径触发卒中分析、仅1路径触发CTP生成”
- [x] 输出状态机（`queued/running/succeeded/failed/cancelled`）
- [x] 输出阶段枚举（`triage/tooling/icv/ekv/consensus/summary/done`）
- [x] 固化字段语义：`available_modalities`、`hemisphere`
- [x] 输出 README 与代码偏差清单

完成标准：
- [x] 团队能用同一套术语沟通（重做交付物：[W1_基线加固交付.md](./W1_基线加固交付.md)）
- [x] 4路径与2触发规则通过真值表校验（验收记录：[W1W2_真值表校验记录.md](./验收记录/W1W2_真值表校验记录.md)）
- [x] 不再争论“当前到底在哪个阶段/状态”（重做交付物：[W1_基线加固交付.md](./W1_基线加固交付.md)）

### 第2周：Tool 契约化
- [x] 冻结 Tool 列表：`detect_modalities/load_patient_context/generate_ctp_maps/run_stroke_analysis/generate_medgemma_report/query_guideline_kb/save_notes`
- [x] 定义每个 Tool 的输入输出和错误码
- [x] 为每个 Tool 新增 `applicability` 与 `precondition`
- [x] 固化路径判定优先级：`mcta+ctp -> mcta -> single-phase -> ncct-only`
- [x] 固化触发规则：  
  `NCCT + mCTA -> generate_ctp_maps -> run_stroke_analysis`  
  `NCCT + mCTA + CTP -> run_stroke_analysis`

完成标准：
- [x] 任意 Tool 都能回答“输入是什么、输出是什么、失败怎么报错”（重做交付物：[W2_Tool契约交付.md](./W2_Tool契约交付.md)）
- [x] 触发矩阵对 4 路径给出唯一执行顺序（验收记录：[W1W2_真值表校验记录.md](./验收记录/W1W2_真值表校验记录.md)）

### 第3周：主链跑通
- [x] 冻结 `Triage Planner` 输入输出协议（交付物：[W3_主链跑通交付.md](./W3_主链跑通交付.md)）
- [x] 固化主链顺序：病例输入 -> 路径判定 -> Tool 调用 -> 结构化结果（交付物：[W3_主链跑通交付.md](./W3_主链跑通交付.md)）
- [x] 定义失败节点重试规则（重试次数、幂等、停止条件）（交付物：[W3_主链跑通交付.md](./W3_主链跑通交付.md)）

完成标准：
- [x] 能从病例输入拿到结构化结果（验收记录：[W3_主链验收记录.md](./验收记录/W3_主链验收记录.md)）
- [x] 失败节点可重试，且可追踪（验收记录：[W3_主链验收记录.md](./验收记录/W3_主链验收记录.md)）

### 第 4 周：ICV（内部一致性）

- [x] 完成 ICV 规则集设计（列出全部规则 ID / status / message）
- [x] 完成 ICV Tool 接入主链（`icv.py` + `_tool_icv` + `report_payload.icv`）
- [x] 在 Processing / Viewer 前端展示 ICV 状态与具体问题列表
- [x] 产出文档：`docs/W4_ICV交付.md` 与 `docs/验收记录/W4_ICV验收记录.md`

完成标准：

- [ ] 用典型病例跑一遍，可在 Viewer 中看到 ICV 总状态 + 至少一条具体问题
- [x] “规则集 + 可视化 + 验收记录”三项均在仓库中有落档（见上两份文档，交付物：[W4_ICV交付.md](./W4_ICV交付.md)，验收记录：[W4_ICV验收记录.md](./验收记录/W4_ICV验收记录.md)）
### 第5周：EKV（外部知识校验）+ Lite 裁决
- [x] 定义 EKV 规则（支持/部分支持/不支持）
- [x] 定义 `Consensus Lite` 启动条件和输出格式
- [x] 定义冲突处理策略（失败/降级/人工复核）

完成标准：
- [x] 关键结论都能被标记校验状态

### 第6周：汇总与证据链
- [ ] 冻结 `FinalReport` 字段
- [ ] 冻结 `EvidenceItem` 与 `AgentEvent` 字段
- [ ] 建立结论到证据的映射关系

完成标准：
- [ ] 报告里的关键结论都能追溯证据来源

### 第7周：前端展示
- [ ] 输出 Agent Cockpit 基础页（步骤、错误、证据、结果）
- [ ] 验证不影响现有 viewer/report/chat

完成标准：
- [ ] 前端可查看一次完整运行轨迹

### 第8周：验收与上线建议
- [ ] 执行功能回归（标准/边界/异常病例）
- [ ] 执行性能与稳定性测试
- [ ] 输出上线建议（上线/灰度/补齐后上线）

完成标准：
- [ ] 形成完整验收报告和风险清单

## 三个必须盯住的里程碑
1. 第3周：系统“能跑”
2. 第6周：结果“可解释”
3. 第8周：版本“可交付”

# W0 状态机与事件字典（冻结版）

## 1. Run 状态（run.status）
- `queued`：已入队，等待执行
- `running`：执行中
- `succeeded`：运行成功结束
- `failed`：运行失败结束
- `cancelled`：被取消
- `paused_review_required`：需人工复核暂停

## 2. Stage 状态（run.stage）
- `triage`
- `tooling`
- `icv`
- `ekv`
- `consensus`
- `summary`
- `done`

## 3. Step 状态（steps[].status）
- `pending`
- `running`
- `completed`
- `failed`
- `skipped`

## 4. 事件类型（events[].event_type）
- `plan_created`
- `step_started`
- `step_completed`
- `issue_found`
- `human_review_required`
- `human_review_completed`
- `writeback_completed`

## 5. W0 映射规则（后端统一）
- `tool_name=triage_planner && status=completed` -> `plan_created`
- `status=running` -> `step_started`
- `status in {completed, skipped}` -> `step_completed`
- `status in {failed, warn, warning}` -> `issue_found`
- `status in {paused_review_required, review_required, await_review}` -> `human_review_required`
- `tool_name` 含 `human_review/human_confirm` 且成功 -> `human_review_completed`
- `tool_name` 含 `writeback` 或 `emr_sync` 且成功 -> `writeback_completed`

## 6. 前端显示建议（W0）
- 状态标签与颜色使用统一 token，不在页面内新增自定义状态。
- 新增字段缺失时，展示 `-` 或占位文案，不抛异常。
